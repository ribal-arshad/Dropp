## Setup (same for all examples)

```ts
import {
  Dropp,
  JsonFileMediaRepository,
  LocalStorageDriver,
  SharpTransformationDriver,
  PluginRegistry,
  WatermarkPlugin,
  AITaggingPlugin,
  SEOPlugin,
} from "droppjs";
import path from "node:path";

const dropp = new Dropp({
  repository: new JsonFileMediaRepository(
    path.join(process.cwd(), ".dropp", "media.json"),
  ),
  storage: new LocalStorageDriver(
    path.join(process.cwd(), "uploads"),
    "/uploads",
  ),
  transformer: new SharpTransformationDriver(),
  plugins: [
    new WatermarkPlugin({
      text: "© My Brand",
      position: "bottomRight",
      opacity: 0.7,
    }),
    new AITaggingPlugin({ model: "default" }),
    new SEOPlugin(),
  ],
});
```

## 1. Basic Attach (upload + store metadata)

Upload a file and store it with metadata tied to a model.

```ts
const media = await dropp.attach({
  file: fileBuffer, // Buffer from multer, fs.readFile, etc.
  fileName: "product-photo.jpg",
  mimeType: "image/jpeg",
  model: "products", // Your database model name
  modelId: "123", // Primary key of the product record
  collection: "gallery", // Optional: group related uploads
  metadata: {
    caption: "Hero product shot",
    photographer: "jane@acme.com",
    license: "internal-use",
  },
});

console.log(media.id); // UUID generated by Dropp
console.log(media.url); // /uploads/products/123/gallery/...
console.log(media.metadata); // Your metadata + AI tags
```

**What happens:**

- File uploaded to storage
- Metadata persisted to repository
- De-duplicated by file hash (same file = no re-upload)
- Plugins run (watermark added, AI tags generated, SEO data extracted)

## 10. Replace Media with Versioning (keep history)

Replace a file but keep the old one as a previous version.

```ts
const v1 = await dropp.attach({
  file: fileBuffer1,
  fileName: "design.psd",
  mimeType: "application/x-photoshop",
  model: "projects",
  modelId: "1",
});

// Designer updates the file
const v2 = await dropp.replace(
  v1.id,
  {
    file: fileBuffer2,
    fileName: "design-v2.psd",
  },
  { version: true }, // Enable versioning
);

console.log(v2.metadata.version); // 2
console.log(v2.metadata.previousVersionId); // v1.id
console.log(v2.metadata.active); // true

// Get the v1 record if you need it
const v1Record = await dropp.get(v1.metadata.previousVersionId);
console.log(v1Record.metadata.active); // false
```

**Versioning behavior:**

- `version: true` → Old record marked inactive, kept as `previousVersionId`
- `version: false` (default) → Old record deleted

## 11. Rollback (revert to previous version)

Restore a previous version after replacement.

```ts
const v2 = await dropp.replace(
  v1.id,
  { file: fileBuffer2, fileName: "design-v2.psd" },
  { version: true },
);

// Oops, v2 is broken. Rollback to v1.
const restored = await dropp.rollback(v2.id);

console.log(restored.id); // v2.id (same media record)
console.log(restored.metadata.version); // 1 (reverted to v1 content)
console.log(restored.metadata.active); // true
```

**Rollback semantics:**

- Takes current media ID
- Restores from `previousVersionId`
- Marks old (current) version as inactive
- Returns reactivated version

## 15. Error Handling

What goes wrong and how to handle it.

```ts
try {
  const media = await dropp.attach({
    file: buffer,
    fileName: "photo.jpg",
    mimeType: "image/jpeg",
    model: "posts",
    modelId: "1",
  });
} catch (error) {
  if (error.message.includes("File too large")) {
    console.log("Upload exceeds size limit");
  } else if (error.message.includes("Invalid MIME type")) {
    console.log("File type not allowed");
  } else if (error.message.includes("Repository error")) {
    console.log("Database/storage write failed");
  } else {
    console.log("Unknown error:", error);
  }
}
```

**Common errors:**

- File too large (storage limit)
- Invalid MIME type (validation)
- Model/modelId missing (required fields)
- Network/storage failure (retry logic)

## 16. Express Integration (full route example)

Bring it together in a real framework.

```ts
import express from "express";
import multer from "multer";
import { Dropp /* ... */ } from "droppjs";

const app = express();
const upload = multer({ storage: multer.memoryStorage() });
const dropp = new Dropp({
  /* setup */
});

// POST /posts/:postId/media
app.post("/posts/:postId/media", upload.single("file"), async (req, res) => {
  try {
    if (!req.file) {
      return res.status(400).json({ error: "No file uploaded" });
    }

    const media = await dropp.attach({
      file: req.file.buffer,
      fileName: req.file.originalname,
      mimeType: req.file.mimetype,
      model: "posts",
      modelId: req.params.postId,
      collection: req.query.collection || "gallery",
      metadata: {
        uploadedBy: req.user.id,
      },
    });

    res.status(201).json({ success: true, media });
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});

// GET /posts/:postId/media
app.get("/posts/:postId/media", async (req, res) => {
  try {
    const allMedia = await dropp.getByModel("posts", req.params.postId);
    res.json(allMedia);
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});

// DELETE /media/:id
app.delete("/media/:id", async (req, res) => {
  try {
    await dropp.delete(req.params.id);
    res.json({ success: true });
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});
```

## 18. Plugin Lifecycle: When Things Happen

Understanding plugin execution order.

```
1. File received (attach called)
2. beforeUpload plugins run ← Watermark applies here
   - Plugin can validate, modify file
   - If error, attach fails
3. File uploaded to storage
4. Metadata written to repository
5. afterUpload plugins run ← AI tagging runs here
   - Plugin can enrich metadata
   - Changes saved back to repository
6. attach() returns with complete media record

Similar flow for delete:
- beforeDelete plugins run ← cleanup hooks
- File deleted from storage
- Record deleted from repository
- afterDelete plugins run (informational)
```

## 19. When to Use What

**Use `attach()`:**

- New file upload
- Adding fresh media to a model
- First time seeing this file

**Use `replace()` without `{ version: true }`:**

- User wants to swap content (old not needed)
- Storage space is critical
- Simple "update" semantics

**Use `replace()` with `{ version: true }`:**

- Content versioning needed (design files, documents)
- Audit trail required
- "Undo" capability expected
- Articles/posts with update history

**Use `rollback()`:**

- User explicitly wants previous version
- Mistake detected after replacement
- Version history is available

**Use `delete()`:**

- Content permanently unwanted
- Model/modelId being deleted
- Storage cleanup

## 26. Batch Operations: Real-World Example

Complete flow: bulk upload gallery, paginate results, cleanup old versions.

```ts
async function manageBlogPostGallery(
  postId: string,
  galleryFiles: Array<{ buffer: Buffer; name: string }>,
) {
  // 1. Upload new gallery images
  console.log("⏳ Uploading gallery...");
  const uploadResult = await dropp.attachMany(
    galleryFiles.map((file) => ({
      file: file.buffer,
      fileName: file.name,
      mimeType: "image/jpeg",
      model: "posts",
      modelId: postId,
      collection: "gallery",
    })),
  );

  console.log(`✓ Uploaded ${uploadResult.successful.length} images`);

  if (uploadResult.failed.length > 0) {
    console.warn(
      `⚠ Failed: ${uploadResult.failed.length}`,
      uploadResult.failed,
    );
  }

  // 2. Fetch paginated gallery
  const page1 = await dropp.list({
    model: "posts",
    modelId: postId,
    collection: "gallery",
    limit: 5,
    offset: 0,
    sort: "createdAt:desc",
  });

  console.log(
    "First 5 images:",
    page1.map((m) => m.fileName),
  );

  // 3. Find old images (older than 7 days)
  const allGallery = await dropp.getByCollection("posts", postId, "gallery");
  const sevenDaysAgo = new Date(Date.now() - 7 * 24 * 60 * 60 * 1000);

  const oldIds = allGallery
    .filter((m) => m.createdAt < sevenDaysAgo)
    .map((m) => m.id);

  // 4. Batch delete old images
  if (oldIds.length > 0) {
    console.log(`🗑️ Deleting ${oldIds.length} old images...`);
    const deleteResult = await dropp.deleteMany(oldIds);
    console.log(
      `✓ Deleted ${deleteResult.deleted}, failed: ${deleteResult.failed.length}`,
    );
  }
}

// Usage
await manageBlogPostGallery("post-1", [
  { buffer: buffer1, name: "hero.jpg" },
  { buffer: buffer2, name: "step1.jpg" },
  { buffer: buffer3, name: "step2.jpg" },
]);
```

| Query + paginate    | `list()`            | `await dropp.list({ model, limit, offset, sort })`     |
| Search by metadata  | `getByMetadata()`   | `await dropp.getByMetadata({ uploadedBy: "user" })`    |
| Delete file         | `delete()`          | `await dropp.delete(mediaId)`                          |
| Delete many         | `deleteMany()`      | `await dropp.deleteMany([id1, id2, id3])`              |
| Replace (hard)      | `replace()`         | `await dropp.replace(id, input)`                       |
| Replace (versioned) | `replace()`         | `await dropp.replace(id, input, { version: true })`    |
| Revert version      | `rollback()`        | `await dropp.rollback(mediaId)`                        |
| With watermark      | Automatic           | Include WatermarkPlugin in setup                       |
| With AI tags        | Automatic           | Include AITaggingPlugin in setup                       |
| Resize/optimize     | `transformations`   | Add to `attach()` options                              |
